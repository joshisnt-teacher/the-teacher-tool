# Exit Ticket Results Page — Pulse

**Date:** 2026-05-28
**App:** Pulse (`the-teacher-tool`)
**Status:** Approved — ready for implementation planning

---

## Overview

Add an exit ticket results page to Pulse so teachers can view student responses, run AI marking, and use AI actions (class analysis, student feedback, at-risk detection) without leaving Pulse. This restores functionality that was removed when Pulse was split from Analytics, and fixes the broken "Results" button on the Exit Tickets page.

The page is **read-focused** and optimised for quick daily use — a teacher finishes an exit ticket, clicks Results, scans who struggled, hits AI Mark All, reads the class summary. No deep analytics or score editing; that stays in Analytics.

## Page Structure

### Header bar
- Breadcrumb: `Dashboard / Exit Tickets / {ticket name}`
- Page title: exit ticket name (from `tasks.name`)
- Class name + run status badge (`Active` / `Closed` / `Draft`) pulled from the task record

### Summary cards (3, side by side)
| Card | Content | Fallback |
|---|---|---|
| Responses | "14 / 26 submitted" (students with any question_result vs total class students) | "0 / N submitted" |
| Class Average | Average `percent_score` across all `question_results` for this task, shown as "71%" | "—" if no results |
| Hardest Question | Question with the lowest average score — "Q3 — 38% avg" | "—" if no results; for confidence checks not applicable (hide card or show "N/A") |

### Tabs
- **Responses** (default)
- **AI Actions**

---

## Responses Tab

A table — one row per student in the class.

### Columns
- **Student** — first + last name, sorted alphabetically by last name
- **Q1, Q2, … Qn** — one column per question on the exit ticket
- **Total** — overall percent score for this student, read from the `results` table (`results.percent_score` where `task_id = assessmentId` and `student_id` matches)

### Per-cell content
- **Written/text answers:** response text (truncated to ~100 chars, expandable) + score badge (`7/10`) + small AI re-mark button (🤖 icon, triggers `useAIMarkResponses` for that single `question_result`)
- **Multiple choice answers:** selected option text + ✓ (correct) or ✗ (incorrect) icon — no score badge needed
- **No response:** grey "—"

### Not-submitted students
Students in the class with no `question_results` for this task appear as a greyed-out row: name + "Not submitted" spanning question columns.

### AI Mark All button
Sits in the Responses card header (top right). Marks all unscored written-answer `question_results` for this task in one call to `useAIMarkResponses`. Shows a spinner + "Marking…" label while pending. Hidden if no OpenAI key is configured (`useOpenAIKeyStatus`).

### What is NOT in this tab
- No editable score inputs
- No add/remove student rows
- No CSV export
- No grade bands (E/P/D/X)

## AI Actions Tab

Uses the existing `ActionsTab` component copied from Analytics into `src/components/assessment/ActionsTab.tsx` in Pulse. No changes to the component itself.

Three actions:
1. **Class Analysis** — summary of strengths, gaps, topics to reteach
2. **Student Feedback** — personalised feedback note per student
3. **Struggling Students** — at-risk list with score and reason

Each action:
- Has a "Run" / "Regenerate" button
- Displays its last-saved result below the button
- Shows a **Download as PDF** button once a result exists (see PDF Export below)

If no OpenAI key is set, the tab shows: "AI actions require an OpenAI API key. Add your key in Settings → AI Marking."

---

## PDF Export

Each AI action result (Class Analysis, Student Feedback, Struggling Students) gets a **Download as PDF** button that appears after the result is generated.

**Implementation:** print-window approach — no new npm dependencies.

On click, open `window.open()` with a new blank tab containing a self-contained HTML document:

```
[Exit ticket name] — [Class name]
[Action title] · Generated [date]
─────────────────────────────────
[AI result content, formatted as readable HTML]
─────────────────────────────────
Generated by Pulse · Edufied
```

Call `window.print()` on the new window immediately after load. The browser's print dialog includes "Save as PDF" on all platforms.

Formatting rules:
- Clean serif or system font, black on white, generous line height
- Summary text as paragraphs, lists as `<ul>/<li>`, student tables as `<table>`
- No sidebar, no nav, no branding chrome — just the content
- A4-friendly margins via `@page { margin: 2cm; }`

This is intentionally minimal. The function can be extracted to a `src/utils/printPdf.ts` helper so it's easy to swap for a proper PDF library later.

## Data queries

| Data | Source | Query key |
|---|---|---|
| Task record (name, class_id, is_exit_ticket) | `tasks` table | `['task', assessmentId]` |
| Class name + status | `classes` table via `task.class_id` | `['class-for-task', assessmentId]` |
| All students in class | `useStudents(classId)` | existing hook |
| Questions for task | `questions` table, ordered by `number` | `['task-questions', assessmentId]` |
| Student overall scores | `results` table (`task_id = assessmentId`) | `['task-results', assessmentId]` |
| Question results (responses + scores) | `question_results` joined to `students` | `['task-question-results', assessmentId]` |
| MC options | `useQuestionOptionsForTask(assessmentId)` | existing hook |
| AI action results | via `useClassActions(assessmentId)` | existing hook |
| OpenAI key status | `useOpenAIKeyStatus()` | existing hook |
